程序是什么？

先别管 C 语言。任何编程语言，第一件事都是：把你想让电脑做的事写清楚。

先建立感觉

电脑很快，但它不会猜

你对人说“帮我煮个泡面”，人可能知道大概怎么做。电脑不行。电脑需要你把步骤写得非常明确。

拿锅
如果你没说拿锅，电脑不会自动脑补。
倒水
如果你没说倒多少水，它也不会猜。
加热
如果你没说什么时候停，它可能一直煮。
一句话记住：程序就是一份写给电脑的精确操作说明书。

代码是什么？

代码就是这份说明书的文字版本。C 语言代码就是用 C 的规则写出来的说明书。

小测：程序最像什么？
2

先让电脑说一句话

这一节只做一件事：让屏幕显示一句话。不要急着理解所有细节。

hello.c
#include <stdio.h>

int main(void) {
    printf("你好，C 语言！\n");
    return 0;
}

先只看最重要的一行

printf("你好，C 语言！\n");

它的意思是：把双引号里的内容显示出来。

printf你可以先理解成“打印到屏幕”。
"你好，C 语言！"双引号里面是要显示的文字。
\n表示换行。不是显示两个字符，而是让光标去下一行。
;分号像一句话的句号，告诉 C：这一句结束了。
那其他几行先怎么理解？

#include <stdio.h> 可以先理解成“我要用显示文字的工具”；main 是程序开始的地方；return 0 表示正常结束。现在不用深挖。

3

为什么需要变量？

先不背定义。我们从一个麻烦开始：如果内容会变，难道每次都手改文字吗？

没有变量的时候

假设你要显示年龄：

写死内容
printf("我今年 18 岁\n");

如果年龄变成 19，你得手动改文字。数据一多，就会很乱。

变量就是有名字的盒子

使用变量
int age = 18;
printf("我今年 %d 岁\n", age);
人话理解：age 是盒子的名字，18 是盒子里的值。后面要用年龄，就用 age 这个名字。

%d 是空位

%d 像一个占位空格，后面的 age 会被填进去。

小测：变量最像什么？
4

为什么数据要分类型？

现在变量已经出现了。下一步才是：盒子也分用途，有的放整数，有的放小数，有的放单个字符。

先看几个不同的数据

18这是整数，适合用 int。
1.75这是小数，适合用 double。
'A'这是一个字符，适合用 char。
types.c
int age = 18;
double height = 1.75;
char grade = 'A';

printf("年龄：%d\n", age);
printf("身高：%.2f\n", height);
printf("等级：%c\n", grade);
类型先这样理解常用打印
int放整数%d
double放小数%f
char放单个字符%c
别急着背范围：刚入门先知道“放什么数据用什么类型”就够了，范围和精度后面再慢慢加。
5

让程序做选择

人会判断：如果下雨就带伞，否则不带。程序也需要这种能力。

if / else：如果……否则……

if_else.c
int score = 75;

if (score >= 60) {
    printf("及格\n");
} else {
    printf("不及格\n");
}

翻译成人话：如果分数大于等于 60，就显示“及格”；否则显示“不及格”。

最容易写错：= 是“赋值”，== 才是“判断相等”。这俩不是一回事。
写法意思
>大于
<小于
>=大于等于
<=小于等于
==判断是否相等
!=判断是否不相等
6

让程序重复做事

打印 100 句话，你不应该写 100 行。重复劳动是电脑的强项。

for：知道重复几次时很好用

for.c
for (int i = 1; i <= 5; i++) {
    printf("第 %d 次：我在学 C\n", i);
}
int i = 1
从 1 开始数。
i <= 5
只要还没超过 5，就继续。
i++
每做完一次，i 增加 1。

while：只要条件成立就继续

while.c
int count = 1;

while (count <= 5) {
    printf("count = %d\n", count);
    count++;
}
死循环：如果条件永远成立，程序会一直跑。循环里通常要有机会改变条件。
7

一个变量不够用怎么办？

现在才讲数组，因为你已经知道变量是一个盒子。数组就是一排同类型盒子。

假设你有 5 个成绩，写 5 个变量很麻烦：

array.c
int scores[5] = {90, 85, 70, 88, 100};

printf("第一个成绩：%d\n", scores[0]);
重点：数组下标从 0 开始。5 个元素的位置是 0、1、2、3、4，不是 1 到 5。

用循环看完整个数组

array_loop.c
for (int i = 0; i < 5; i++) {
    printf("scores[%d] = %d\n", i, scores[i]);
}
8

重复代码太多怎么办？

这时候再讲函数。函数就是把一段代码打包成一个小工具，想用时叫它一下。

function.c
int add(int a, int b) {
    return a + b;
}

int result = add(3, 5);
函数名add，这个工具叫什么。
参数a 和 b，给工具的输入。
返回值return 后面的结果。
人话理解：函数像一个小机器，把 3 和 5 放进去，它吐出 8。
9

一组信息想放在一起怎么办？

现在才讲结构体。因为你已经知道变量和数组了，结构体就是把不同类型的信息打包。

一个学生不只有一个成绩，还可能有姓名、年龄、分数。它们应该属于同一个“学生”。

struct.c
struct Student {
    char name[20];
    int age;
    double score;
};

struct Student s1 = {"Tom", 18, 92.5};

s1.age 的意思就是：s1 这个学生里面的 age。

10

最后再碰指针：地址的纸条

指针一开始很吓人，是因为它直接碰到了内存地址。现在你已经有一点基础，可以轻轻碰一下。

变量住在内存里

你可以把内存想象成很多小房间。变量是房间里的东西，地址是房间门牌号。

变量房间里的东西。
地址房间的门牌号。
指针一张写着门牌号的纸条。
pointer.c
int x = 10;
int *p = &x;

printf("x = %d\n", x);
printf("*p = %d\n", *p);
先这样记：&x 是 x 的地址；p 存着地址；*p 是顺着地址找到值。
为什么危险：如果纸条上的地址错了，程序可能找错地方，甚至改坏别的数据。

小练习：不要只看，要动手

每学一点，写一个十几行的小程序。C 语言不是看会的，是敲会的。

练习 1：打印自我介绍输出姓名、年龄、喜欢什么。
练习 2：年龄判断输入年龄，判断是否成年。
练习 3：1 到 100 求和用循环累加。
练习 4：成绩列表用数组存几个成绩，求平均值。

示例：猜数字

guess.c
#include <stdio.h>

int main(void) {
    int answer = 42;
    int guess;

    while (1) {
        printf("请输入你猜的数字：");
        scanf("%d", &guess);

        if (guess > answer) {
            printf("太大了\n");
        } else if (guess < answer) {
            printf("太小了\n");
        } else {
            printf("猜对了\n");
            break;
        }
    }

    return 0;
}

GitHub 托管方式

终极版建议用 3 个文件：index.html、manifest.json、sw.js。这样手机添加到桌面更像 App，也能支持基础缓存。

推荐结构：仓库根目录放 index.html、manifest.json、sw.js 三个文件。GitHub Pages 打开后，手机 Safari / Chrome 可以添加到桌面。
项目结构
c-language-guide/
├── index.html
├── manifest.json
└── sw.js
创建仓库
GitHub 右上角 + → New repository，名字例如 c-language-guide。
上传文件
把这个 HTML 保存为 index.html，再把 manifest.json 和 sw.js 一起上传到仓库根目录。
开启 Pages
进入 Settings → Pages，选择 Deploy from a branch，分支选 main，目录选 /(root)。
手机打开
访问 https://你的用户名.github.io/c-language-guide/，然后添加到主屏幕。
更新检查说明：页面会每次打开请求 GitHub 上最新的 index.html，从里面读取版本号。如果 GitHub 上版本更高，会提示“发现更新”。sw.js 负责基础缓存和 Service Worker 更新。
